Database Technologies

Database Technologies

Overview

The Esoclusty E-Learning Platform employs a hybrid database approach, combining the strengths of MongoDB for flexible document storage and MySQL for structured relational data. This section explores the database architecture, configuration, and optimization strategies.

MongoDB 4.4+ - Primary Database

Document-Based Architecture

MongoDB serves as the primary database for the platform, providing:

Flexible Schema Design

  • Dynamic Schema: Easy adaptation to changing data requirements
  • Document Structure: Natural representation of complex data
  • Embedded Documents: Efficient storage of related data
  • Array Support: Native support for list-based data structures

Use Cases in E-Learning

  • Course Content: Rich multimedia content with flexible structure
  • User Profiles: Comprehensive user information and preferences
  • Learning Progress: Complex progress tracking and analytics
  • Content Metadata: Flexible tagging and categorization

Performance & Scalability Features

Indexing Strategies

  • Single Field Indexes: Basic performance optimization
  • Compound Indexes: Multi-field query optimization
  • Text Indexes: Full-text search capabilities
  • Geospatial Indexes: Location-based features support

Sharding & Distribution

  • Horizontal Scaling: Distribution across multiple servers
  • Shard Key Selection: Optimal data distribution strategy
  • Balancing: Automatic data rebalancing across shards
  • Zone-based Sharding: Geographic and logical data distribution

Memory Management

  • WiredTiger Storage Engine: Advanced compression and performance
  • In-Memory Storage: Fast access to frequently used data
  • Cache Management: Intelligent memory allocation
  • Compression: Reduced storage requirements

MySQL 8.0+ - Relational Database

ACID Compliance & Transactions

MySQL provides structured data storage with ACID guarantees:

Transaction Support

  • ACID Properties: Atomicity, Consistency, Isolation, Durability
  • Transaction Isolation Levels: Configurable isolation for performance
  • Deadlock Detection: Automatic deadlock resolution
  • Rollback Capabilities: Transaction rollback and recovery

Use Cases in E-Learning

  • User Authentication: Secure user credentials and sessions
  • Course Enrollments: Structured enrollment and progress data
  • Comments & Ratings: Relational data for user interactions
  • System Metadata: Configuration and system state information

Performance Optimization

Query Optimization

  • Query Planner: Intelligent query execution planning
  • Index Optimization: Strategic index creation and maintenance
  • Query Caching: Result caching for repeated queries
  • Connection Pooling: Efficient connection management

Storage Engines

  • InnoDB: ACID-compliant storage engine with row-level locking
  • MyISAM: High-performance read operations
  • Memory: In-memory tables for temporary data
  • Archive: Compressed storage for historical data

Hybrid Database Strategy

Data Distribution Strategy

Strategic data placement optimizes performance and functionality:

MongoDB Data

  • Course Content: Rich multimedia and structured content
  • User Profiles: Flexible user information and preferences
  • Learning Analytics: Complex progress and performance data
  • Content Metadata: Flexible categorization and tagging

MySQL Data

  • User Accounts: Authentication and authorization data
  • Course Enrollments: Structured enrollment relationships
  • Comments & Reviews: User-generated content and feedback
  • System Configuration: Application settings and metadata

Data Synchronization

Consistency Management

  • Eventual Consistency: Acceptable consistency for most operations
  • Synchronous Operations: Critical operations with immediate consistency
  • Data Validation: Cross-database data integrity checks
  • Conflict Resolution: Handling of data conflicts and inconsistencies

Performance Benefits

  • Optimized Queries: Database-specific query optimization
  • Reduced Complexity: Simpler data models for specific use cases
  • Scalability: Independent scaling of different data types

MySQL Table Design

User Management Tables

-- Users table for authentication
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role ENUM('ADMIN', 'TEACHER', 'STUDENT') NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_email (email),
    INDEX idx_role (role)
);

-- User profiles for additional information
CREATE TABLE user_profiles (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    avatar_url VARCHAR(500),
    bio TEXT,
    preferences JSON,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_user_id (user_id)
);

Course Management Tables

-- Course enrollments
CREATE TABLE course_enrollments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    course_id VARCHAR(255) NOT NULL, -- MongoDB ObjectId as string
    enrollment_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    progress DECIMAL(5,2) DEFAULT 0.00,
    status ENUM('ACTIVE', 'COMPLETED', 'PAUSED') DEFAULT 'ACTIVE',
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY unique_enrollment (user_id, course_id),
    INDEX idx_user_id (user_id),
    INDEX idx_course_id (course_id),
    INDEX idx_status (status)
);

-- Comments and ratings
CREATE TABLE course_comments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    course_id VARCHAR(255) NOT NULL,
    comment TEXT NOT NULL,
    rating INT CHECK (rating >= 1 AND rating <= 5),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_course_id (course_id),
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at)
);
