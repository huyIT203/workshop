Environment Setup

Overview

Before we can start building the Esoclusty E-Learning Platform, we need to set up a proper development environment. This section covers the installation and configuration of all required tools and software.

Prerequisites Installation

Java Development Kit (JDK 17+)

Download & Installation

# Download JDK 17 from Oracle or use OpenJDK
# Oracle JDK: https://www.oracle.com/java/technologies/downloads/
# OpenJDK: https://adoptium.net/

# For Ubuntu/Debian
sudo apt update
sudo apt install openjdk-17-jdk

# For CentOS/RHEL
sudo yum install java-17-openjdk-devel

# For macOS (using Homebrew)
brew install openjdk@17

# For Windows
# Download installer from Oracle website and run

Environment Configuration

# Set JAVA_HOME environment variable
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64
export PATH=$JAVA_HOME/bin:$PATH

# Add to ~/.bashrc or ~/.zshrc for persistence
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

Verification

# Verify Java installation
java -version
javac -version

# Expected output:
# openjdk version "17.0.x" 2023-xx-xx
# OpenJDK Runtime Environment (build 17.0.x+xx-xx)
# OpenJDK 64-Bit Server VM (build 17.0.x+xx-xx, mixed mode, sharing)

Maven 3.9+

# Extract to /opt
sudo tar -xzf apache-maven-3.9.6-bin.tar.gz -C /opt
sudo mv /opt/apache-maven-3.9.6 /opt/maven

# Set environment variables

export PATH=$PATH:$MAVEN_HOME/bin

# Add to ~/.bashrc for persistence
echo 'export MAVEN_HOME=/opt/maven' >> ~/.bashrc

MongoDB 4.4+

Installation Methods

Ubuntu/Debian Installation

# Import MongoDB public GPG key
wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -

# Create list file for MongoDB
echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/4.4 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.4.list

# Update package database
sudo apt update

# Install MongoDB
sudo apt install -y mongodb-org

# Start MongoDB service
sudo systemctl start mongod
sudo systemctl enable mongod

# Verify installation
sudo systemctl status mongod

CentOS/RHEL Installation

# Create MongoDB repository file
sudo tee /etc/yum.repos.d/mongodb-org-4.4.repo << EOF
[mongodb-org-4.4]
name=MongoDB Repository
baseurl=https://repo.mongodb.org/apt/redhat/\$releasever/mongodb-org/4.4/x86_64/
gpgcheck=1
enabled=1
gpgkey=https://www.mongodb.org/static/pgp/server-4.4.asc
EOF

# Install MongoDB
sudo yum install -y mongodb-org

# Start and enable service
sudo systemctl start mongod
sudo systemctl enable mongod

macOS Installation

# Using Homebrew
brew tap mongodb/brew
brew install mongodb-community@4.4

# Start MongoDB service
brew services start mongodb-community@4.4

Windows Installation

Troubleshooting Common Issues

Java Installation Issues
# Problem: java command not found
# Solution: Check PATH and JAVA_HOME
echo $PATH
echo $JAVA_HOME

# Problem: Wrong Java version
# Solution: Update PATH order
which java
ls -la /usr/bin/java
Maven Installation Issues
# Problem: mvn command not found
# Solution: Check MAVEN_HOME and PATH
echo $MAVEN_HOME
echo $PATH

# Problem: Maven can't find Java
# Solution: Ensure JAVA_HOME is set correctly
echo $JAVA_HOME
MongoDB Issues
# Problem: MongoDB won't start
# Solution: Check logs and permissions
sudo tail -f /var/log/mongodb/mongod.log
sudo chown -R mongodb:mongodb /var/lib/mongodb
sudo chown -R mongodb:mongodb /var/log/mongodb

# Problem: Can't connect to MongoDB
# Solution: Check bindIp configuration
sudo nano /etc/mongod.conf
# Ensure bindIp: 127.0.0.1 or 0.0.0.0
MySQL Issues
# Problem: MySQL won't start
# Solution: Check logs and permissions
sudo tail -f /var/log/mysqld.log
sudo chown -R mysql:mysql /var/lib/mysql

# Problem: Access denied for user
# Solution: Reset root password or recreate user
sudo mysql -u root
ALTER USER 'root'@'localhost' IDENTIFIED BY 'new_password';

Performance Optimization

System Tuning
# Increase file descriptor limits
echo "* soft nofile 65536" | sudo tee -a /etc/security/limits.conf
echo "* hard nofile 65536" | sudo tee -a /etc/security/limits.conf

# Optimize MongoDB settings
sudo nano /etc/mongod.conf
# Add/modify:
# operationProfiling:
#   mode: slowOp
#   slowOpThresholdMs: 100

# Optimize MySQL settings
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf
# Add/modify:
# innodb_buffer_pool_size = 1G
# max_connections = 200
